Write pros and cons for merge and rebase. Describe situations for preferably usage of these commands.
Git Merge:
Плюси:
    Простий у використанні та розумінні.
    Зберігає історію комітів гілки.
    Безпечний, оскільки він не змінює існуючу історію, отже, не руйнує посиланняМінуси:
    Може призвести до складної, нелінійної історії через численні злиття гілок.
Git Rebase:
Плюси:
   -  Забезпечує чітку та лінійну історію проекту шляхом інтеграції змін із базової гілки в тематичну гілку.
   -  Rebase дозволяє легко скасувати опцію за допомогою команди rebase -i.
Мінуси:
   - Може стати складним вирішення конфліктів оскільки треба вирішувати кожний коміт
